# 회고
캐시와 테이터베이스의 정합성을 확인하기 위해 직접 데이터를 비교했다. 만약 부하 테스트 도중에 캐시와 데이터베이스 사이에 발생하는 불일치 문제를 어떻게 파악할 수 있을지 고민이 된다.
기존에 완성된 채팅 시스템을 Transactional Outbox Pattern을 도입하여 도중에 Redis를 종료하는 방식으로 장애 대응을 테스트 했다. 이것도 실무에서 어떻게 테스트 하는지 궁금해졌다.

전반적으로 부하 테스트를 위한 환경 셋팅에 익숙해진 것을 느꼈다.
그런데 테스트 측정 결과를 어떻게 파악해야 할지 더 학습해야 할 것 같다.
